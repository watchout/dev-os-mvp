# Grok × dev-OS コンテンツ作成ワークフロー

## 概要

Grokを使って投稿文を作成し、dev-OSのカレンダーで管理・自動投稿するワークフロー。

---

## ワークフロー

```
┌─────────────────┐
│  1. Grokで作成   │  ← プロンプトテンプレートを使用
└────────┬────────┘
         ↓
┌─────────────────┐
│  2. カレンダー追加 │  ← content/calendar/2026-01.yml
└────────┬────────┘
         ↓
┌─────────────────┐
│  3. 投稿実行     │  ← npm run marketing:x:post
└─────────────────┘
```

---

## 1. Grokへのプロンプトテンプレート

### 📝 週間投稿バッチ作成

```
あなたはAI開発ツール「dev-OS」のマーケティング担当です。

【プロダクト概要】
- AI開発の品質管理OS
- 「AIで作ったコード、結局自分で直してませんか？」という課題を解決
- SSOT（Single Source of Truth）で仕様を整え、AIの出力品質を安定させる
- ターゲット: スタートアップのCTO/テックリード

【今週の投稿を作成してください】
以下の4タイプを各1つずつ作成:

1. **Build in Public** (開発進捗・学び)
   - 開発中に発見したこと、失敗談、学びを共有
   - 人間味のある語り口

2. **Tips** (Cursor/AI開発のTips)
   - 具体的で再現可能なTips
   - 「すぐ使える」価値

3. **バズ狙い** (問いかけ・共感)
   - 開発者の痛みに刺さる問いかけ
   - 「わかる〜」と思わせる

4. **思想系** (AI開発の未来・哲学)
   - 深い洞察や考察
   - 議論を呼ぶ内容

【フォーマット】
- 各投稿は280文字以内
- ハッシュタグは2-3個
- 絵文字は適度に（1-2個）

【NGワード】
- 「革命」「画期的」など大げさな表現
- 宣伝色の強い表現
```

---

### 🔥 バズ狙い投稿作成

```
あなたはAI開発ツール「dev-OS」のマーケティング担当です。

【プロダクト概要】
dev-OSは「AIで作ったコード、結局自分で直してませんか？」という課題を解決するAI開発品質管理OSです。

【今話題のトピック】
（ここにトレンドや話題を入力）

【依頼】
このトピックに関連させて、開発者の共感を呼ぶ投稿を3パターン作成してください。

条件:
- 280文字以内
- 問いかけ形式 or 「あるある」形式
- 押し付けがましくない
- dev-OSの宣伝は入れない（共感獲得が目的）
```

---

### 💡 Tips投稿作成

```
あなたはAI開発ツール「dev-OS」のマーケティング担当です。

【今週学んだこと/発見したこと】
（ここに内容を入力）

【依頼】
これを元に、開発者向けのTips投稿を作成してください。

条件:
- 280文字以内
- 具体的で再現可能
- 「へぇ、使ってみよう」と思わせる
- Before/After があると◎
```

## 2. カレンダーへの追加方法

### 方法A: 直接編集

`content/calendar/2026-01.yml` を開いて追加:

```yaml
  "2026-01-04":
    weekday: "土"
    x:
      - time: "10:00"
        type: "tips"
        content: |
          Grokで作成した投稿文をここに貼り付け
          
          #AI開発 #Tips
        status: "scheduled"
```

### 方法B: コマンドで追加

```bash
npm run marketing:x:schedule -- "投稿内容" "2026-01-04T10:00:00+09:00"
```

---

## 3. 投稿実行

### 即時投稿
```bash
npm run marketing:x:post -- "投稿内容"
```

### スケジュール確認
```bash
npm run marketing:check -- upcoming
```

## 週次ルーティン

### 日曜日（30分）

1. **Grokで来週の投稿を一括作成**
   - 週間バッチ作成プロンプトを使用
   - 6-7投稿分を作成

2. **カレンダーに追加**
   - `content/calendar/2026-01.yml` を編集

3. **確認**
   ```bash
   npm run marketing:check -- week
   ```

### 毎日（5分）

1. **朝: 予定確認**
   ```bash
   npm run marketing:check
   ```

2. **投稿時間: 実行**
   ```bash
   npm run marketing:x:post -- "内容"
   ```

## Tips

### Grokへの追加指示例

- 「もっとカジュアルに」
- 「具体例を入れて」
- 「問いかけ形式で」
- 「140文字以内に短縮して」
- 「トレンドの○○に絡めて」

### 投稿のバリエーション

同じテーマでも切り口を変える:
- 事実ベース → 感情ベース
- 問題提起 → 解決策提示
- 一般論 → 具体例
- テキストのみ → スレッド形式
