# IYASAKA dev-OS | UI/UX & URL Path Specification

## 1. デザイン原則：IYASAKA の「整える」UI

IYASAKA ブランド・マスター・プロトコルに基づき、以下の原則を UI 全体に適用します。

*   **「プラス」のUI:** 開発者の既存の思考を妨げず、必要な時にそっと知恵をプラスする。
*   **「ミエル」の視覚化:** 「今どのフェーズか」「AIが何に基づいて考えているか（SSOT）」を常に可視化し、確信を与える。
*   **「体温」のライティング:** 機械的なメッセージではなく、現場の伴走者としての言葉を選ぶ。

---

## 2. URL パス設計（サイトマップ）

### 2.1 公開エリア (Landing Page)
主に集客とブランドメッセージの伝達を担当します。

| パス | 画面名 | 内容・役割 |
| :--- | :--- | :--- |
| `/` | トップ（LP） | dev-OS のナラティブ（不から光へ）の提示、CTA |
| `/features` | 機能紹介 | SSOT 管理、多重監査、ガバナンスの詳細説明 |
| `/pricing` | 料金プラン | 各プラン（Solo 〜 Enterprise）の比較とウェイトリスト |

### 2.2 認証エリア (Web Console)
開発の「コクピット」として機能します。

| パス | 画面名 | 内容・役割 |
| :--- | :--- | :--- |
| `/login` | ログイン | Supabase Auth による認証 |
| `/register` | 新規登録 | ユーザー登録 |
| `/console` | ダッシュボード | 所属組織一覧、最近のワークスペース、クレジット残量 |
| `/console/:org` | 組織トップ | 組織内のワークスペース一覧、メンバー一覧 |
| `/console/:org/:ws` | ワークスペース詳細 | **中核画面**。知恵の設計図（Plan）、SSOT プレビュー、実行 |
| `/console/:org/:ws/history` | 実行履歴 | 過去の Plan 実行ログ、Halt 理由の確認 |
| `/console/:org/:ws/ssot` | SSOT エディタ | ワークスペース固有の SSOT 編集・管理 |
| `/settings/profile` | プロフィール設定 | ユーザー情報、通知設定 |
| `/settings/:org` | 組織設定 | プラン変更、Stripe 連携、APIキー（BYO）管理 |

## 3. 画面構成設計 (Layout & Components)

### 3.1 共通レイアウト：コクピット・フレーム
*   **Sidebar (Navigation):** 組織/ワークスペースの切り替え、主要機能へのショートカット。
*   **Header (Status):** 現在のコンテキスト（組織/WS名）、クレジット残量、ユーザーアイコン。
*   **Main Content:** 各画面の固有コンテンツ。

### 3.2 ワークスペース詳細画面（メイン画面）
開発者が最も時間を過ごす画面です。「確信」を生むための 3 カラム構成を推奨します。

1.  **左カラム：知恵の源泉 (SSOT Explorer)**
    *   現在参照している SSOT（`features.yml`, `platform.yml` 等）のツリー表示。
2.  **中央カラム：整える場 (Execution & Plan)**
    *   目的（Purpose）とモード（Mode）の選択。
    *   **知恵の足跡 (Steps Timeline):** AI がどのようなステップで思考するかを可視化。
3.  **右カラム：成果の鍵 (Preview & Export)**
    *   生成されたプロンプト、AI ガイドのプレビュー。
    *   Cursor 等へのコピーボタン。

## 4. インタラクション・ガイドライン

*   **Halt 表示:** ガバナンスにより停止した際、単なる「エラー」ではなく「設計の破綻を未然に防ぎました」という安堵感を与えるメッセージを表示する。
*   **Loading:** AI が思考中（実行中）は、「知恵を整えています...」といった体温のあるマイクロコピーとアニメーションを用いる。
*   **Empty States:** まだ SSOT が定義されていない場合、「ここから知恵を整え始めましょう」と促すガイドを表示する。
