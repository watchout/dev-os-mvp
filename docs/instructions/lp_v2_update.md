# IYASAKA dev-OS | LP 改修指示書 (v2.0)

## 1. 目的
マーケティング監修（2026-01-02）に基づき、LP のキャッチコピー、ターゲット訴求、および CTA をアップデートし、PLG（Product-Led Growth）モデルに最適化する。

## 2. 改修内容

### 2.1 ファーストビュー (Hero Section)
*   **ヘッドライン変更**:
    *   旧: 「AI開発の不透明を確信へ」
    *   新: **「AIで作ったコード、結局自分で直してませんか？」**
*   **サブテキスト変更**:
    *   「『なんとなく作る』から『正本（SSOT）を整え、確実に実装する』へ」という文言を残しつつ、**「孤軍奮闘するリーダーのための AI 開発品質管理プラットフォーム」** であることを強調。

### 2.2 CTA (Call to Action)
*   **文言変更**:
    *   旧: 「無料で相談する」
    *   新: **「無料で確信を手に入れる（Freeプラン）」** または **「今すぐ無料で始める」**
*   **リンク先**:
    *   `https://iyasaka.co.jp/contact?p_id=dev-os&utm_campaign=free_start` へ誘導（将来的に `/register` へ直結）。

### 2.3 料金プランセクション (Pricing)
*   `ssot/pricing.yml` に基づき、プラン表示を 4 段階に更新。
    1.  **Free (¥0)**: 「まずは触ってみる」。1ユーザー/1ワークフロー月10回。
    2.  **Pro (¥4,980)**: 「本気の個人開発に」。無制限実行、多重監査（Reviewer）。
    3.  **Team (¥29,800〜)**: 「チームで品質を統一」。シートライセンス、Linear/Plane 連携。
    4.  **Enterprise (¥200,000〜)**: 「企業レベルのガバナンス」。SSO、監査ログ。

### 2.4 ペイン訴求 (Pain Points)
*   `ssot/personas.yml` の「タカシさん（CTO）」のペインを前面に出す。
    *   「AIに任せると結局自分で直す手間が増える」
    *   「メンバー間の品質のバラツキ」
    *   「仕様のブラックボックス化」

### 2.5 デモ動画プレースホルダー
*   ヒーローセクション下部に、将来デモ動画を埋め込むための `<div class="demo-video-placeholder">` を追加。
*   初期状態では「Coming Soon」またはコンセプトを示す GIF アニメーションを配置し、視覚的な引きを作る。

## 3. ガバナンス・ブランド規約
*   **「不」を「光」へ**: 現場の溜息（手直しコスト）を確信に変える物語を一貫させる。
*   **プラスの UI**: 派手なエフェクトより、清潔感と「整っている」印象を重視。

## 4. 対象ファイル
*   `apps/lp/index.html`
*   `apps/lp/style.css` (必要に応じて)
