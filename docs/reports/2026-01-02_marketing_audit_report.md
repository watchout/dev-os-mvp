## 🔄 重要な変更内容

### 1. 料金プラン再設計

**旧プラン（v1）→ 新プラン（v2）マッピング:**

```
Solo (¥10,000)     → Pro (¥4,980)      # 価格ダウン、個人向け最適化
Starter (¥30,000)  → Team (¥29,800)    # シートライセンス方式に変更
Growth (¥100,000)  → Team              # 統合
Studio (¥250,000)  → Enterprise        # 統合
Enterprise (¥1M〜) → Enterprise (¥200,000〜)
                   + Free (¥0)         # 新規追加（PLG必須）
```

**設計AIへの依頼:**
- [ ] `apps/platform` のプラン関連コードで `plan_id` enum を参照している箇所の洗い出し
- [ ] マイグレーション戦略の検討（既存ユーザーがいる場合）

### 2. ペルソナ定義

**プライマリペルソナ: 「孤軍奮闘CTOのタカシさん」**

```yaml
demographics:
  role: "CTO / テックリード"
  team_size: "4〜8人"
  budget_authority: "月額10万円まで自己決裁可能"

pain_points:
  1. "AIに任せると、直すのに結局自分が入らないといけない" ← 最重要
  2. "メンバーにAI使わせると、品質がバラバラになる"
  3. "仕様が頭の中にしかなくて、引き継ぎが怖い"
  4. "SaaS基本要件を毎回漏らす"
```

**設計AIへの依頼:**
- [ ] LP・UIのコピーライティング時は `ssot/personas.yml` のペイン優先順位を参照
- [ ] 機能優先度を決める際はペルソナの `goals` と `decision_criteria` を考慮

## ⚠️ 設計AIへの要確認事項

### 1. 無料プラン実装の技術影響

`ssot/pricing.yml` で定義した Free プランの制限:

```yaml
limits:
  users: 1
  workspaces: 1
  workflow_executions_per_month: 10
  ssot_files: 5
  execution_log_retention_days: 7
  api_key_slots: 1
```

**確認依頼:**
- 上記制限を実装するために `platform.yml` のエンティティ・機能に追加が必要か？
- `execution_log` のリテンション制御は現在の設計で対応可能か？

### 2. シートライセンス（Team プラン）

Team プランは基本5シート + 追加シート（¥5,980/月）方式。

**確認依頼:**
- `organization_member` テーブルでシート数カウントは可能か？
- 課金連携（Stripe）でシート数による従量課金は対応可能か？

### 3. LP改修の優先順位

`ssot/marketing.yml` に LP 改善計画を記載。優先度:

1. **Critical:** 無料トライアルCTA追加
2. **High:** ファーストビュー改修（ヘッドライン変更）
3. **High:** デモ動画追加
4. **Medium:** FAQ セクション追加

**確認依頼:**
- `apps/lp/index.html` の改修は実装AIに依頼してよいか？
- ブランドプロトコル（`IYASAKA_BRAND_PROTOCOL.md`）との整合性チェックは設計AI側で行うか？

## 🔖 変更根拠

本レポートの変更は、以下のマーケティング原則に基づいています：

1. **PLGファースト**: B2B DevTools は「まず触ってもらう」が命
2. **ニッチ支配戦略**: 広く浅くではなく、狭く深く支配してから拡大
3. **ペイン優先順位**: 最も切迫感のある課題から訴求
4. **価値階段の明確化**: Free→Pro→Team→Enterprise の各段階の価値差分を明示
5. **LTV最大化**: 累積価値によるロックイン、チャーン対策の設計
